I. Open Source

Karpathy's AutoResearch: The Autonomous ML Research Loop

Andrej Karpathy — former Tesla AI lead, OpenAI co-founder, and coiner of "vibe coding" — released autoresearch, a minimalist 630-line Python script that automates the scientific method for machine learning. The agent reads its own source code, forms hypotheses, modifies training parameters, runs experiments within a fixed compute budget (typically 5 minutes on a GPU), evaluates results, and iterates — all without human involvement.

126 Experiments overnight
~700 Changes in 2 days
11% Efficiency gain
8.6M+ Views on X

In one overnight run, the agent drove validation loss from 0.9979 to 0.9697. Over two days of continuous operation, it processed approximately 700 autonomous changes, finding roughly 20 additive improvements that transferred to larger models — cutting the "Time to GPT-2" leaderboard metric from 2.02 hours to 1.80 hours. Karpathy noted the agent caught attention scaling and regularization oversights he had missed in two decades of manual work.

"Seeing the agent do this entire workflow end-to-end and all by itself… is wild." — Andrej Karpathy

The reaction was swift. Varun Mathur, CEO of Hyperspace AI, distributed the loop across a peer-to-peer network. On the night of March 8–9, 35 autonomous agents ran 333 experiments completely unsupervised:

  • Hardware diversity as strategy — H100 GPUs used brute force while CPU-only agents got creative, focusing on initialization and normalization strategies they could test within tighter compute budgets
  • Gossip-based discovery — When one agent found Kaiming initialization dropped loss by 21%, the discovery spread network-wide within hours via GossipSub protocol; 23 agents incorporated it into their own hypotheses
  • Compressed history — In 17 hours, agents independently rediscovered ML milestones like RMSNorm and tied embeddings — breakthroughs that took human researchers at Google Brain and OpenAI roughly eight years to formalize

Strategic Implication: This demonstrates AI-driven research loops applicable far beyond ML. Any domain involving hypothesis → experiment → evaluate cycles — marketing optimization, drug discovery, materials science — could benefit from this paradigm. The open-source MIT license makes enterprise adoption frictionless.

II. Enterprise

Microsoft 365 E7: The $99 "Frontier Suite" for the Agentic Enterprise

Microsoft announced Microsoft 365 E7, its highest-tier enterprise subscription at $99 per user per month, available May 1, 2026. Dubbed the "First Frontier Suite built on Intelligence + Trust," E7 bundles the full E5 stack with AI agent management and advanced security into a single license.

$99 Per user / month
13.2% Bundle discount
$15 Agent 365 standalone
May 1 GA date

The bundle includes Microsoft 365 E5, Microsoft 365 Copilot, the new Agent 365 control plane for governing AI agents, Microsoft Entra Suite for identity management, and advanced Defender, Intune, and Purview security capabilities covering both human users and AI agents.

Agent 365 is the genuinely new piece — a governance layer for AI agents operating across the enterprise, handling access controls, monitoring, and lifecycle management. It reaches general availability on May 1 at $15/user/month as a standalone product.

"Gartner believes organizations will find the value of ME7 to be questionable for the majority of knowledge workers today… Upgrading to the ME7 bundle for Agent 365 is not advised until Microsoft adds value." — Gartner Analysis, March 2026

Gartner found the 13.2% bundle discount underwhelming compared to historical E3→E5 discounts, and called Agent 365 "a work in progress with limited net new functionality." They warned enterprises to avoid nonreduction clauses in contracts, in case Microsoft revises the offerings due to low uptake.

Strategic Implication: Microsoft is positioning for an "agentic enterprise" future where AI agents are licensed like employees. The market signal is clear, but the product may be ahead of enterprise readiness. A wait-and-see approach is advised unless your organization is already deep in the Microsoft AI ecosystem.

III. Developer Tools

Anthropic Code Review: Multi-Agent Bug Detection for Claude Code

Anthropic launched Code Review, a multi-agent system integrated into Claude Code that automatically analyzes pull requests for logic bugs before human reviewers see the code. Available in research preview for Claude for Teams and Claude for Enterprise customers.

3× Review feedback gain
$2.5B+ Claude Code run-rate
4× Enterprise subs YTD

The problem is straightforward: Claude Code has dramatically increased code output, creating a bottleneck at the pull request review stage. As Cat Wu, Anthropic's Head of Product, put it: "Now that Claude Code is putting up a bunch of pull requests, how do I make sure those get reviewed in an efficient manner?"

The system integrates with GitHub and auto-triggers on PR creation. Multiple agents analyze the codebase in parallel from different perspectives — one might focus on control flow, another on data handling. A final aggregation agent deduplicates, ranks, and prioritizes findings. The focus is strictly on logic errors over style, with step-by-step reasoning that explains what each issue is, why it's problematic, and how to fix it.

Issues are classified by severity:

Critical — must fix Potential — worth reviewing Historical — preexisting issues

Engineering leads can customize additional checks based on internal best practices. Target customers include Uber, Salesforce, and Accenture — companies already using Claude Code at scale. Anthropic is also navigating a legal dispute with the Department of Defense over a supply chain risk designation, making its enterprise revenue line increasingly important.

Strategic Implication: As AI-generated code volume explodes, automated quality assurance becomes critical infrastructure. Anthropic is building the full pipeline — code generation (Claude Code) → review (Code Review) → security analysis (Claude Code Security) — creating significant switching costs for enterprise customers already in the ecosystem.

IV. AI Infrastructure

Nvidia NemoClaw: Open-Source AI Agent Platform for the Enterprise

Nvidia is building an open-source platform for enterprise AI agents, internally referred to as NemoClaw, set to be announced at GTC (its annual developer conference in San Jose) next week. The platform will allow companies to deploy AI agents across their workforces — and critically, it will work regardless of whether those companies run Nvidia hardware.

Open Source Licensing model
5+ Enterprise partners pitched
GTC '26 Expected launch

Nvidia has been pitching NemoClaw to major enterprise software companies including Salesforce, Cisco, Google, Adobe, and CrowdStrike to forge launch partnerships. Partners would likely receive early access in exchange for contributing to the project. Nvidia plans to include security and privacy tools as a core component — directly addressing the concerns that have made enterprise adoption of autonomous agents controversial.

Some tech companies, including Meta, have asked employees to refrain from using AI agents on work computers due to unpredictability and security risks. A Meta employee who oversees safety and alignment publicly shared a story about an agent going rogue and mass-deleting her emails. — WIRED, March 9, 2026

The platform connects to Nvidia's existing Nemo model framework and represents a significant strategic shift. Nvidia's software strategy has historically been built on CUDA, a proprietary system that locks developers into building for Nvidia GPUs. NemoClaw being chip-agnostic and open source signals that Nvidia sees the agent layer as a new competitive surface — one where ecosystem breadth matters more than hardware lock-in.

Separately, Nvidia is expected to reveal a new inference computing chip system at GTC that incorporates a chip designed by Groq, following a multibillion-dollar licensing agreement from late last year.

Strategic Implication: Nvidia is making its play for the AI agent infrastructure layer — open source, chip-agnostic, enterprise-grade security. If NemoClaw gains adoption, Nvidia becomes the platform layer between AI models and enterprise workflows, maintaining relevance even as companies like Google, Amazon, and Meta build custom silicon. The CUDA moat protected hardware; NemoClaw aims to build a software moat for agents.

V. Strategy

AI for Boring Businesses: Four Lessons from Main Street

Sam Gerstenzang, former Stripe product leader and partner at incubator Boulton and Watt, argues that the real AI opportunity isn't in building the next SaaS darling — it's in operationally complex, real-world service businesses. His portfolio includes Meadow (a funeral home that uses wedding venues during the day) and Moxie (a platform for nurses starting aesthetic medicine clinics, which just raised a $25M Series C). The thesis: as software gets easy for anyone to build and public SaaS companies are down 70%+, resilient businesses with operational moats will win.

Four hard-won lessons from bringing AI to Main Street:

  • Research compressed from 3 months to 3 weeks — AI at every step: ChatGPT for market sizing and margin analysis, Clay for lead enrichment and outbound, Claude Code to spin up landing pages in an hour, Granola + Claude to extract insights from customer calls. Each pivot costs less, moves faster.
  • Reward outcomes, not AI usage — "AI initiatives" fell flat when they became checkbox activities. The danger: team members outsourcing judgment to "the hallucinations of a madman." The breakthrough came when an analyst used Claude Code to build an entirely new interactive map for customer density data — not just cleaning spreadsheets. Concrete examples of transformation beat mandates every time.
  • AI customer acquisition is just another channel — Generative Engine Optimization is the new SEO, and AI-generated email outreach is flooding inboxes. But OpenAI started selling ads on ChatGPT, and Google search revenue is still up 17% YoY. Treat AI channels like any new channel: test, measure, don't assume paradigm shift.
  • Humans remain stubbornly hard to replace — In operationally complex businesses (funeral homes, medical clinics), the human element isn't a weakness to optimize away — it's the value proposition. AI augments the humans; it doesn't replace the relationships, judgment, and regulatory navigation that these businesses run on.
"The use of AI itself is not the goal — using AI only matters if it can improve the quality and the speed of the work. Hallucinations are your problem, not the LLM's." — Sam Gerstenzang, Boulton and Watt

Strategic Implication: This is the Changeis playbook in a nutshell. Federal consulting is the definition of an "operationally complex, regulated, real-world service business." The firms that integrate AI into how they deliver — not just what they deliver — will outpace competitors who treat AI as a bolt-on. The lesson on rewarding outcomes over AI usage is directly applicable to any federal team rolling out AI tools: mandate results, not methods.
